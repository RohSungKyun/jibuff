End-to-end flow for a jibuff request. Phase 1 (Interview): user submits a natural language requirement; InterviewEngine conducts a Socratic Q&A using LLM scoring until ambiguity/risk pass thresholds, then generates spec/tasks.md. Phase 2 (Agent Loop): LoopController picks tasks one by one, spawns the agent CLI, runs validators, and optionally runs the quality evaluator (rtc mode). On validator/quality failure the task is requeued with failure context for the next iteration.
